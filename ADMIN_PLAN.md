# תוכנית דשבורד מנהלים ומערכת תחרויות

## סטטוס נוכחי
- **Users** – קיים (users)
- **Competitions** – כרגיל כ־tournaments (סכום + שם + נעילה)
- **Bets/Entries** – submissions (ניחושים + סטטוס + תשלום)
- **Transactions** – אין טבלה ייעודית; paymentStatus בתוך submissions
- עמלה 12.5% – מחושבת בקוד (FEE_PERCENT)
- הרשאות – role: admin/user/agent

---

## שלב 1 (יישום ראשון) ✅
- [x] טבלת **site_settings** (מפתח–ערך): טקסטים, צבעים, URL באנר
- [x] הרחבת **tournaments**: תיאור, סוג (תחרויות ספורט/לוטו/צ'אנס), תאריכי התחלה/סיום, מקס משתתפים, חלוקת פרסים (JSON)
- [x] דשבורד מנהל ראשי: סיכום סטטיסטיקות, רשימת תחרויות עם כספים, כניסה לטפסים/משחקים/שחקנים
- [x] יצירת תחרות: שם, תיאור, סוג, סכום, תאריכים, מקס משתתפים, חלוקת פרסים (מקום 1 / 3 מקומות / מותאם)
- [x] דוחות כספיים: סה"כ כניסות, עמלה 12.5%, סכום לפרסים – בממשק הניהול

## שלב 2 (המשך)
- [ ] עריכת כל טקסט באתר מתוך דשבורד (שימוש ב־site_settings בממשק)
- [ ] עריכת תמונות ובאנרים (שמירת URL או path ב־site_settings)
- [ ] עריכת צבעים ועיצוב בסיסי (CSS vars או שמירה ב־site_settings)
- [ ] פתיחה/סגירה גלובלית של הרשמה (מפתח site_settings: registration_open)

## שלב 3 – סוגי תחרויות
- [ ] **תחרויות ספורט**: בחירת ליגה או הזנת משחקים ידנית, טבלת ניחושים אוטומטית (כבר קיים מודל matches)
- [ ] **לוטו/צ'אנס**: הגדרת כמות מספרים, טווח (1–37), מספר זוכים; טבלת הגרלות

## שלב 4 – טכני

## טבלאות DB (מצב נוכחי + שלב 1)

| טבלה | שימוש |
|------|--------|
| users | משתמשים, תפקידים |
| tournaments | תחרויות (מורחב: תיאור, סוג, תאריכים, מקס משתתפים, חלוקת פרסים) |
| matches | משחקים (תחרויות ספורט) |
| submissions | טפסים/הגשות (Entries) |
| agent_commissions | עמלות סוכנים |
| **site_settings** | מפתח–ערך: טקסטים, צבעים, באנרים |

---

## הרשאות
- **ADMIN**: יצירה/עריכה של תחרויות, עריכת הגדרות אתר, צפייה בדוחות, רשימת משתמשים, אישור/דחיית טפסים.
- **User**: הרשמה והשתתפות בתחרויות פתוחות בלבד.
